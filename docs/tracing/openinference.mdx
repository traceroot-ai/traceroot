Traceroot is built on OpenTelemetry and supports standard semantic conventions for AI observability.

## OpenInference

[OpenInference](https://github.com/Arize-ai/openinference) defines semantic conventions for AI/ML observability. Traceroot recognizes these attributes:

| Attribute | Mapping |
|-----------|---------|
| `openinference.span.kind` | Maps to Traceroot span kinds (`LLM`, `AGENT`, `TOOL`; `CHAIN` → `SPAN`) |
| `llm.token_count.prompt` | Input token count |
| `llm.token_count.completion` | Output token count |
| `llm.token_count.total` | Total token count |

## GenAI Semantic Conventions

Traceroot also supports the [GenAI semantic conventions](https://opentelemetry.io/docs/specs/semconv/gen-ai/) from the OpenTelemetry project:

- `gen_ai.system` — the LLM provider (e.g., `openai`, `anthropic`)
- `gen_ai.request.model` — the model name
- `gen_ai.usage.input_tokens` / `gen_ai.usage.output_tokens`

## Span Kind Detection

Traceroot determines span kind using this priority order:

1. **Explicit** — `traceroot.span.type` attribute (set via `@observe(type=...)`)
2. **OpenInference** — `openinference.span.kind`
3. **GenAI** — inferred from GenAI semantic convention attributes
4. **Heuristic** — inferred from other attributes (e.g., presence of a model name suggests `llm`)

## Interoperability

Because Traceroot uses OpenTelemetry under the hood, traces from other OTEL-compatible tools can be ingested and displayed. OpenInference and GenAI conventions ensure that span kinds and token counts are correctly interpreted regardless of the source.

This means you can use Traceroot alongside other OpenTelemetry-based instrumentation without conflicts.
