---
title: Bring Your Own Key (BYOK)
description: Use your own LLM API keys to power the Traceroot AI agent
---

By default, the Traceroot AI agent uses system-provided models. With BYOK, you can use your own API keys to control which model powers the agent.

## Default System Models

Without BYOK, these models are available out of the box:

| Provider | Models |
|----------|--------|
| Anthropic | Claude Opus 4.6, Sonnet 4.6, Sonnet 4.5, Haiku 4.5 |
| OpenAI | GPT-5, GPT-5 Mini, o3, o4-mini |

## Supported Providers

| Provider | Description |
|----------|-------------|
| OpenAI | GPT models via OpenAI API |
| Anthropic | Claude models via Anthropic API |
| Azure OpenAI | GPT models via Azure-hosted endpoints |
| Google | Gemini models via Google AI |
| Amazon Bedrock | Models via AWS Bedrock |
| DeepSeek | DeepSeek models |
| OpenRouter | Access multiple providers via OpenRouter |

## Setup

1. Go to **Workspace Settings → Model Providers** in the dashboard
2. Select a provider from the list
3. Enter your API key
4. (Optional) Set a custom base URL for self-hosted models
5. Save — the key is encrypted at rest

Keys are scoped to your workspace. Each workspace can have its own provider configuration.

## Custom Base URLs

For self-hosted or proxied models, you can set a custom base URL per provider. This is useful for:

- **Self-hosted open-source models** behind an OpenAI-compatible API
- **Corporate proxies** that route API traffic through internal endpoints
- **Regional API endpoints** for data residency requirements

## Selecting a Model

Once a provider is configured, you can select which model to use in the AI chat interface. The model selector shows both system models and any models available through your BYOK configuration.
