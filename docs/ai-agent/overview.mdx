Traceroot includes a built-in AI agent that goes beyond dashboards. It can analyze your traces, clone your repository into a sandboxed environment, identify root causes, and create fix PRs — all directly from the trace viewer.

## The Problem

Observability tools show you *what* went wrong. You still have to figure out *why* and *how to fix it*. That means context-switching between your dashboard, your IDE, your git history, and your terminal.

## The Solution

Traceroot's AI agent closes that loop. When you see a failing trace, you can ask the agent to:

- **Analyze the trace** — summarize errors, identify patterns, compare latency across spans
- **Find the root cause** — clone your repo, read the relevant source code, trace the execution flow
- **Fix the bug** — write a fix in a sandboxed environment and create a PR on GitHub

## Agent Tools

The agent has access to these tools:

| Tool | Description |
|------|-------------|
| `query_traces` | Search and filter traces with aggregation |
| `download_trace` | Export full trace data (spans, hierarchy, metadata) |
| `check_github_access` | Verify GitHub App access to a repository |
| `git_clone` | Clone a repository into the sandbox |
| `bash` | Execute commands in the sandbox |
| `read` / `write` | Read and write files in the sandbox |

## Architecture

```
Trace (in ClickHouse)
  → AI Agent (LLM-powered)
    → Sandbox (Docker or Daytona)
      → GitHub PR
```

The agent runs in an isolated sandbox environment. It cannot access your production systems — only the trace data and a clone of your repository.
