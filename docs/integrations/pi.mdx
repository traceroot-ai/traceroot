Trace [pi](https://pi.dev) coding-agent sessions to TraceRoot with the
`@traceroot-ai/pi-extension` package. pi is a minimal coding-agent harness and does not
emit OpenTelemetry natively, so the extension is the integration surface: it subscribes
to pi's lifecycle events and exports a span tree — sessions, turns, LLM calls, and tool
executions — to TraceRoot over OTLP.

## Setup

Install the extension with pi's package manager:

```bash
pi install npm:@traceroot-ai/pi-extension
```

Enable it and point it at TraceRoot:

```bash
export TRACEROOT_ENABLED=true
export TRACEROOT_API_KEY=<your-api-key>
```

Tracing is opt-in — no spans are emitted unless `TRACEROOT_ENABLED` is set. For a local
TraceRoot instance, add `TRACEROOT_LOCAL_MODE=true`.

## Usage

Run pi as usual. Every session is traced automatically:

```bash
pi
```

Send a prompt and the agent's work is captured as a nested trace:

```
pi.session
└── pi.turn                       # one per prompt
    ├── openai/gpt-4o-mini        # LLM call: tokens, cost
    │   ├── pi.tool.read          # tool: arguments + result
    │   └── pi.tool.bash
    └── openai/gpt-4o-mini        # second turn, after tool results
```

In the TUI, a `Traceroot ●` status and a clickable trace URL appear in the footer. The
`/traceroot` command controls tracing at runtime: `status`, `open`, `flush`, `disable`,
and `enable`.

## What Gets Captured

| Attribute | Description |
|-----------|-------------|
| Session | One span per pi session (the trace root) |
| Turn | One span per prompt / agent loop |
| Model | Provider and model id for each LLM call |
| Messages | Request messages and the assistant response, on every span |
| Tool calls | Each tool execution with arguments, result, and error flag |
| Tokens | Input, output, total, and cache read/write counts |
| Cost | Per-call cost from the model's reported usage |
| Context usage | Context-window tokens and percentage |
| HTTP status | Provider response status, with rate-limit events |
| Latency | Duration per span |

Tool spans are parallel-safe and nest under the LLM call that requested them.

## Run the example

Install the extension, enable it, and run any prompt that uses tools — for example,
*"Read package.json, edit the description field, then run `git diff` and tell me what
changed."* The full session → turn → LLM → tool tree appears in TraceRoot.
