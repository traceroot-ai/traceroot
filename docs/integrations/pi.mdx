Trace [pi](https://pi.dev) coding-agent sessions you embed in your own Node process using
the `@traceroot-ai/pi` package. `@earendil-works/pi-coding-agent` is the library form of
pi — you drive an `AgentSession` yourself instead of running the interactive CLI —
and `@traceroot-ai/pi` patches `AgentSession.prototype` so every session is traced with
full agent/LLM/tool span semantics, no manual span code required.

## Setup

Install the instrumentation alongside the SDK it patches:

```bash
npm install @traceroot-ai/pi @earendil-works/pi-coding-agent
```

Instrument before creating any session:

```typescript
import * as pi from '@earendil-works/pi-coding-agent';
import { instrumentPiCodingAgent } from '@traceroot-ai/pi';

// Instrument BEFORE creating any session — instrumentPiCodingAgent patches
// AgentSession.prototype.
instrumentPiCodingAgent(pi, {
  apiKey: process.env.TRACEROOT_API_KEY,
});
```

`@earendil-works/pi-coding-agent` is ESM-only and requires Node >=22.19.

## Usage

Create and run a session as usual — every session created after instrumentation is
traced automatically:

```typescript
import { AuthStorage, createAgentSession, ModelRegistry } from '@earendil-works/pi-coding-agent';

const authStorage = AuthStorage.create();
const modelRegistry = ModelRegistry.create(authStorage);
const model = modelRegistry.find('openai', 'gpt-4o-mini');

const { session } = await createAgentSession({ model, authStorage, modelRegistry });
try {
  await session.prompt('Write add.js exporting add(a, b), then write and run a test for it.');
} finally {
  session.dispose();
}
```

Each `session.prompt()` call is captured as a nested trace:

```
AgentSession.prompt              # AGENT span — one per prompt() call
└── gpt-4o-mini                  # LLM span — provider, tokens, cost
    ├── write: add.js            # TOOL span — arguments + result
    ├── write: test-add.js
    └── bash: node test-add.js
```

Tool spans nest under the LLM span that requested them.

## What Gets Captured

| Attribute | Description |
|-----------|-------------|
| Agent runs | One span per `session.prompt()` call |
| Model | Model id for each LLM call |
| Messages | Prompt and assistant response, when content capture is enabled |
| Tool calls | Each tool execution with arguments, result, and error status |
| Tokens | Input, output, and cache read/write counts |
| Cost | Per-call cost from the model's reported usage |
| Latency | Duration per span |

`captureContent` and `captureToolIo` (both default `true`) control whether prompt/response
text and tool arguments/results are attached to spans; set either to `false` to keep that
content off exported spans.
